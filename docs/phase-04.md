# Phase 04 — Angular frontend

## Outcome

A user can use the wallet in a browser.

## Use cases

- View wallet balance and history.
- Submit deposit, withdrawal, and transfer forms.
- See API errors in the UI.

## User stories

- As a user, I want to view my wallet in a dashboard.
- As a user, I want to perform wallet actions from the browser.

## Functional requirements

- Create the Angular app shell and routing.
- Add an API service layer.
- Add dashboard, wallet details, forms, and transaction history UI.
- Show loading and error states.

## Non-functional requirements

- The UI must be responsive.
- Forms must be readable and accessible.
- Components should stay reusable.
- State handling should stay simple.

## Executable code tasks

- Scaffold the Angular application with routing.
- Build an API client service for wallet requests.
- Add the dashboard and wallet details pages.
- Add forms for deposit, withdraw, and transfer with validation.
- Add a transaction history table with loading and error states.

## Done when

- A user can view a wallet and perform actions from the UI.
