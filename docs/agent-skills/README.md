# Agent skills

These docs are reusable operating rules for implementation agents working in this repository.

## When to read them

- Before coding a Spring Boot slice.
- Before coding an Angular slice.
- Before coding a fullstack slice that touches both.
- Before writing Bruno contract tests.
- When a phase document needs an explicit system logic or style rule.

## Files

| File | Focus |
| --- | --- |
| [llm-coding-style.md](./llm-coding-style.md) | Style index / compatibility router |
| [llm-coding-style-frontend.md](./llm-coding-style-frontend.md) | Angular frontend coding style and UI/state rules |
| [llm-coding-style-backend.md](./llm-coding-style-backend.md) | Spring Boot backend coding style and core business reminders |
| [llm-coding-style-fullstack.md](./llm-coding-style-fullstack.md) | Cross-layer Angular + Spring Boot coordination |
| [bruno-testing.md](./bruno-testing.md) | Bruno testing conventions and API scenarios |

## Rule of thumb

Phase files describe what to build. Agent skills describe how to build it.
